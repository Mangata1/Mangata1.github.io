Federated Learning: Strategies for Improving Communication Efficiency

发表于2021-05-06|更新于2021-05-07|FL
|字数总计:687|阅读时长:4分钟|阅读量:

Motivation

Because of low user connection reliability and communication transmission speed slowly , the communication step between server and client devices is likely to the bottle-neck of Federated Learning. The paper propose some methods to reduce the uplink communication costs. There are two kind of update. Structured update and sketch update.

○ Structured Updates: learn a restricted space parametrized using a smaller number of variables.

○ Sketched Updates: learn a full model,and then compress it using a combination of quantization, random rotations and subsampling.

Optimization Method

Let’s talk about structured updates,we got two ways in structured update.

one is low rank, The first type of communication efficient update restricts the updates H to have a pre-specified structure. It is worth mentioning that the matrix H ,the paper only say it is a $d_1$ rows and $d_2$ columns matrix .We enforce every update to local model H to be a low rank matrix of rank at most k, where k is a fixed number. In order to do so, we express H the product of two matrix : $H=A *B$ where A is $d_1$ rows and k columns matrix,B is k rows and $d_2$ columns matrix. A is a matrix generated by random seeds ,so we can compress it into a random seed. So the clients only need to send matrix B to the server. If not, the paper how calculate a matrix B to make A multiply B have the matrix H most information .

The other is random mask, My personal understanding is that the mechanism of random mask is similar to the dropout layer of CNN . Generate a zero one matrix using random seed, the client only send random seed and the H matrix part of weights where the position of zero one matrix is equal one. All in one word ,we discard part of the data with a certain probability.

Sketched Updates

Sketched update is another way to address communication cost. It also has two way in sketched update, subsampling and probabilistic quantization.

•subsampling: only communicate matrix $H^‘$which is formed from random subset of the values of $H$ . Server will average it. This can be done so that the average of the sampled updates is an unbiased estimator of the true average.

•Probabilistic quantization: compress the updates is by quantizing the weights.

After checking the information, I found that the method of Probabilistic quantization from another paper:: BinaryConnect: Training DNN with binary weights during propagations, binaryConnect can compress weight from Thirty-two bytes to four bytes float . But when max = 1 and min = −1 one and most of values are 0, the 1-bit quantization will lead to a large error. So,the paper improve the quantization by structured random rotations. We note that applying a random rotation on h before the quantization (multiplying h by a random orthogonal matrix) solves this issue .My personal understanding is that multiplying h by a orthogonal matrix means Convert the vector from the current coordinate system to the coordinate system represented by the orthogonal matrix .

Experiment

low rank and mask comparsion

Get random mask perform significantly better than low rank, so we can omit the low rank in other comparison.
Accuracy seems to be unaffected with reduced update size in random mask.

Compare random mask and rotation

Random mask learn better than sketch update(blue and green line) which contain subsampling and rotation, but sketch update attain modest accuracy faster.
Since sketch update will throw part of parameters away, random mask will have better accuracy in the end.
Moreover, model perform better with rotation.

Compare all sketched update without structured update

Random rotation improve the performance, particular in the small number of quantization bits and smaller modes.
With sketch out all but 6.25 and quantize to 2 bits ,we only get minor drop in convergence.

Sketch update (no random mask): Reddit data

With random rotation will performance better.
Quantization into 2 bits will not cause any loss in performance.(1 bits may cause)
Only subsampling 50 client without even touch other 1950 clients once.

Effect of number of client: Reddit data

it means More client, less communication. equal More communication, less client.
